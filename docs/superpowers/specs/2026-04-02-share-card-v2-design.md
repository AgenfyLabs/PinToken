> 日期: 2026-04-02
> 状态: 待确认

---

## 1. 概述

分享卡片是 PinToken 的核心增长引擎。V2 目标：**让用户看到卡片就想截图发出去**。

架构：背景模板（静态 PNG）+ 数据覆盖层（Canvas 渲染动态数字/图表）

用户可在 5 套皮肤中切换，每套皮肤 = 一张背景图 + 一组坐标/色值配置。

---

## 2. 尺寸与格式

- 画布尺寸：**1080 × 1080 px**（正方形，适配 X / 微信 / Instagram）
- 输出格式：PNG（下载）/ JPEG（分享链接，体积更小）
- Retina 支持：Canvas 内部 2x 渲染（2160 × 2160），CSS 尺寸 1080 × 1080

## 4. 五套皮肤

### 4.1 Terminal（终端暗夜）

- **背景**：纯 #0d1117，顶部微弱的橙色光晕（径向渐变）
- **字体色**：#e6edf3（主文字）、#FF6B35（橙色高亮）、#27c93f（绿色/省钱）
- **特征**：等宽字体、左上角三个交通灯圆点（●●●）、边框用细线 #30363d
- **适合**：硬核开发者

### 4.2 Neon（霓虹闪烁）

- **背景**：纯黑 #000000，中央橙色发光体（高斯模糊圆形渐变，不透明度 15%）
- **字体色**：#ffffff（主）、#FF6B35 带 glow 效果（text-shadow 模拟）、#00ff88（绿色 neon）
- **特征**：数字有微弱发光效果（双层绘制：底层大号模糊 + 上层清晰）、条形图有荧光感
- **适合**：喜欢酷炫视觉的

### 4.3 Minimal（极简白）

- **背景**：#fafafa，底部 1/3 淡灰 #f0f0f0 色块
- **字体色**：#1a1a1a（主）、#FF6B35（橙色）、#16a34a（绿色）
- **特征**：无衬线字体（系统默认 sans-serif）、大量留白、无边框、条形图用圆角矩形
- **适合**：设计师型开发者、浅色 feed 中更醒目（反差）

### 4.4 Gradient（渐变冲击）

- **背景**：从左下到右上的渐变 #1a0a2e → #2d1b69 → #FF6B35（紫到橙）
- **字体色**：#ffffff（全白）、rgba(255,255,255,0.6)（次要文字）
- **特征**：超大号数字、半透明白色卡片容器（backdrop 效果）、条形图纯白
- **适合**：社交媒体用户、Spotify Wrapped 感

### 4.5 Blueprint（工程蓝图）

- **背景**：#0a192f，叠加细网格线（每 40px，#172a45，0.3 不透明度）
- **字体色**：#64ffda（青绿主色）、#ccd6f6（浅蓝灰）、#FF6B35（橙色数据点）
- **特征**：等宽字体、数据区域用虚线框、条形图用描边而非填充
- **适合**：基础设施 / DevOps 工程师

## 6. 数据元素详细

### 6.1 英雄数字区

| 元素 | 字号 | 色值 | 格式 |
|------|------|------|------|
| "本月花费" 标签 | 18px | textDim | 静态文字 |
| 花费金额 | **72px bold** | accent (#FF6B35) | $XXX.XX |
| "省了" 标签 | 20px | textDim | 静态文字 |
| 省钱金额 | **36px bold** | saving (#27c93f) | $XX.XX |
| 省钱百分比 | 24px | saving | ↓ XX% |

### 6.2 统计条（四格）

| 格子 | 数值格式 | 说明 |
|------|---------|------|
| API 调用 | 1,847 calls | 本月请求数 |
| Token 总量 | 2.4M tokens | input + output |
| Provider 数 | 5 providers | 使用的 provider 数量 |
| 追踪天数 | 30 天 | 从首次记录到现在 |

### 6.3 模型分布条形图

- 最多显示 **4 个模型**（按花费 DESC）
- 第 5 个及以后合并为 "Others"
- 每行：模型名（左对齐）+ 条形图（中间）+ 百分比（右对齐）
- 条形图宽度 = 百分比 × 最大宽度（约 500px）
- 条形图高度 = 28px，圆角 4px
- 条形图颜色 = barFill，背景 = barBg

## 7. Dashboard 交互

### 7.1 分享弹窗改版

```
┌─────────────────────────────────┐
│  生成分享卡片                     │

│                                  │

│  [Terminal] [Neon] [Minimal]     │

│  ─────────────────────────        │

└─────────────────────────────────┘
```

### 7.2 交互行为

| 操作 | 行为 |
|------|------|
| 点击皮肤名 | 实时切换预览（重新渲染 Canvas） |
| 下载图片 | 导出 1080×1080 PNG，文件名 `pintoken-2026-03.png` |
| 复制图片 | 调用 Clipboard API 复制 PNG 到剪贴板（fallback: 提示下载） |
| 关闭 | ESC 或点击遮罩关闭 |

### 7.3 皮肤记忆

用 localStorage 记住用户上次选择的皮肤。

## 8. 技术实现

### 8.1 文件结构

```
dashboard/
├── share-card.js          # 重写：Canvas 渲染引擎 + 皮肤系统
├── share-card-skins.js    # 5 套皮肤的配置数据
├── backgrounds/           # 预渲染的背景图（可选，复杂背景用）
│   └── (空目录，所有背景纯代码绘制)
└── index.html             # 分享弹窗 HTML 结构
```

### 8.2 渲染流程

```
1. 获取数据（/api/summary + /api/models + /api/providers）
2. 根据选中皮肤读取配置
3. 创建 2160×2160 Canvas（Retina 2x）
4. 绘制背景层（纯色/渐变/网格等，根据皮肤配置）
5. 绘制数据层（数字/文字/条形图，坐标固定）
6. 输出到预览 <canvas>（缩放到 360×360）
7. 导出时输出 1080×1080 PNG
```

### 8.3 关键 API

需要新增或复用的 API 端点：

| 端点 | 返回 | 用途 |
|------|------|------|
| `/api/summary` | 已有 | 今日/昨日花费、累计节省 |
| `/api/month-summary` | 已有 | 本月花费、token、请求数 |
| `/api/models` | 已有 | 模型使用分布 |
| `/api/providers` | 已有 | provider 数量 |
| `/api/share-data` | **新增** | 聚合以上所有数据为分享卡片专用格式 |

`/api/share-data` 返回格式：

## 9. 不做的事情（YAGNI）

- 不做云端分享 URL（M2 后期或 M3）
- 不做排名估算（数据不足时宁可不做）
- 不做成就徽章（分享卡片 V3）
- 不做自定义颜色编辑器（5 套预设够用）
- 不做动画/视频分享（只做静态图片）
